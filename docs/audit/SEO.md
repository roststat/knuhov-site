## H. Техническое SEO

1. **robots.txt — нет.** Яндекс и Google индексируют вслепую; нет указателя на sitemap. **P0.**
2. **sitemap.xml — нет.** Сайт молодой, без карты Яндекс будет узнавать о страницах медленно. Правильно — генерировать в `build.js` из списка страниц + `SERVICES` (единый источник правды). **P0.**
3. **h1: есть только на / и /diagnostika.** Все остальные страницы (uslugi, contacts, cases, case-1/2 и все 11 usluga-*) открываются с `<h2>` — шаблон `servicePageMain()` в `build.js:184` печатает `<h2>${s.title}</h2>`. Один h1 на страницу — база. **P0.**
4. **canonical — нет нигде.** Шаблон `HEAD` (`build.js:126-139`) его не печатает. При наличии зеркал/UTM/`#`-вариантов — риск дублей. Добавить в шаблон с передачей пути страницы. **P0.**
5. **Open Graph / Twitter cards — нет.** Ссылку на сайт шарят лично (тёплый трафик из ТЗ!) — превью в Telegram/WhatsApp сейчас пустое. Это не «просто SEO», это упаковка главного канала. og:title/description/url/image + twitter:card. **P0.**
6. **Favicon/apple-touch-icon/manifest — нет.** Вкладка безликая, в выдаче Яндекса фавикон — элемент CTR. Мозаика бренда — готовый мотив для SVG-фавикона. **P0.**
7. **Чистота URL:** все ссылки — `/index.html`, `/uslugi.html#...`. `vercel.json` отсутствует → `cleanUrls` выключен. Включить `cleanUrls` + переписать внутренние ссылки на `/`, `/uslugi`, `/usluga-crm` и т.д. Vercel сам 308-редиректит `.html`-варианты. **P1** (до подачи sitemap — чтобы в индекс сразу пошли чистые URL).
8. **404:** кастомной нет — Vercel отдаёт стандартную. Сделать `404.html` в стиле сайта с навигацией. **P1.**
9. **Дубли:** `/case-1.html` и `/case-2.html` идентичны (title, description, контент) — классический дубль. Пока кейсов нет — оставить одну страницу-заглушку или закрыть обе от индексации (`noindex`), вернуть при публикации реальных кейсов. **P1.**
10. **title/description:** на основных страницах уникальные и осмысленные; у страниц услуг title = `«{title} — Кнухов консалтинг»`, description = teaser (коротковато, без ключей «для клиники/медцентра»). Переписать по формуле из CONVENTIONS. У contacts description «Контакты и реквизиты Кнухов консалтинг.» — слабый. **P1.**
11. **lang="ru"** есть (`build.js:127`) — ок. hreflang не нужен (один язык/регион).
12. **Внутренняя перелинковка:** страницы услуг не ссылаются ни на каталог, ни на смежные услуги, ни на диагностику. Хлебных крошек нет. Добавить: крошки (+BreadcrumbList), блок «Смежные услуги» из `SERVICES` той же категории, CTA на диагностику. **P1.**
13. Шрифты/скорость — влияют на ранжирование через CWV, см. AUDIT.md §F.

## I. Структурированные данные Schema.org

Сейчас — ноль разметки. Внедрять JSON-LD в шаблон `build.js` (не руками в готовые .html):

1. **Organization** (`ProfessionalService`) + **Person** (Ислам Кнухов, founder) + **WebSite** — на все страницы через шаблон HEAD/footer. **P1.**
2. **Service** — на каждую `/usluga-*` страницу, генерировать из объекта `SERVICES` (name=title, description=teaser, provider=Organization). Бесплатно масштабируется на новые услуги. **P1.**
3. **BreadcrumbList** — вместе с UI-крошками (Главная → Услуги → {категория} → {услуга}). **P1.**
4. **FAQPage** — после добавления FAQ-блоков (диагностика, услуги). **P2.**
5. **WebSite + SearchAction** — поиска на сайте нет; не размечать (Яндексу не врать). Если появится поиск — добавить.
6. LocalBusiness с адресом — только после реальных реквизитов/адреса от владельца.
