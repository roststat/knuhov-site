# HOW-TO-RUN — запуск одного шага в новой сессии

Скопируй текст ниже в новую сессию Claude Code (открытую в репозитории `~/Downloads/knuhov-site`), подставив номер шага.

---

Выполни ОДИН под-шаг плана: docs/plan/steps/S__-*.md (подставь точное имя файла).

Порядок работы:
1. Сначала прочитай docs/plan/STATE.md — убедись, что зависимости шага в статусе done, а сам шаг не занят/не выполнен. Если зависимости не выполнены — остановись и скажи, чего не хватает.
2. Прочитай файл шага целиком, затем его секцию «Контекст-бутстрап» и ВСЕ файлы из списка «прочитай первыми» (как минимум: docs/plan/README.md, docs/plan/CONVENTIONS.md, docs/design-guide.md, build.js).
3. Поставь шагу статус doing в docs/plan/STATE.md.
4. Сделай только этот шаг, точно по его секции «Точные инструкции». Соблюдай docs/plan/CONVENTIONS.md и docs/design-guide.md. Корневые *.html руками не редактируй — только src/, партиалы и build.js, затем `node build.js`.
5. Проверь по секциям «Definition of Done» и «Как проверить локально» этого шага.
6. Закоммить результат одним коммитом `Sxx: краткая суть` (на русском).
7. Обнови docs/plan/STATE.md: статус done (или blocked с причиной), заметка, при необходимости — строка «Следующий шаг» и запись в Журнал.
8. Остановись. НЕ деплой и НЕ пушь в origin. Деплой делается только по моей отдельной команде «пуш» (= git push origin main + vercel --prod --yes).

Если по ходу нашёл проблему вне рамок шага — не чини её, а допиши одной строкой в Журнал STATE.md как кандидата в новый шаг.

---

## Команда «пуш» (отдельная сессия или конец сессии, только по явной команде владельца)

```
cd ~/Downloads/knuhov-site && git push origin main && vercel --prod --yes
```
