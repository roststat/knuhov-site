# План трансформации knuhov.ru

Пакет создан 2026-07-04 по итогам аудита ([../audit/AUDIT.md](../audit/AUDIT.md), [../audit/SEO.md](../audit/SEO.md), чеклист — [../audit/CHECKLIST.md](../audit/CHECKLIST.md)).

**Цель:** «супер-сайт», но не лендинг — многостраничность и глубина; изображения в хиро и на карточках; аккуратная анимация; сильное SEO (Яндекс + Google). Бренд остаётся строго ч/б (Space Grotesk, фирменный нижний-правый угол); все правила — в [CONVENTIONS.md](CONVENTIONS.md).

## Как этим пользоваться

- Один шаг = одна сессия с чистым контекстом. Готовый промт для запуска — [HOW-TO-RUN.md](HOW-TO-RUN.md).
- Источник правды о прогрессе — [STATE.md](STATE.md) (статусы, зависимости, «следующий шаг»).
- Каждый шаг — самодостаточный файл `steps/Sxx-*.md`: контекст, зависимости, точные инструкции, Definition of Done, проверка.
- Деплой — только по команде владельца «пуш» (= commit + `git push origin main` + `vercel --prod --yes`).

## Легенда

**Фазы:** A — SEO-фундамент и техбаза · B — Медиа и дизайн · C — Контент и страницы · D — Анимация · E — Аналитика, формы, юридическое.
**Статусы (в STATE.md):** `todo` / `doing` / `done` / `blocked(причина)`.
**«Данные владельца»** — шаг требует информации от Ислама; без неё выполняется подготовительная часть, шаг остаётся `blocked`.
