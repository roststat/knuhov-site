# S18 — Реальные контакты, соцсети и реквизиты

## 1. Цель
Убрать публично видимые заглушки «[Черновик: …]» со страницы контактов и `href="#"` из соцссылок футера — прямой удар по доверию на сайте, который продаёт доверие.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (консалтинг для клиник Ислама Кнухова). Плоский HTML/CSS/JS, сборка `node build.js`. Заглушки: `src/pages/contacts.html` — телефон/email/соцсети (~строки 12–25) и реквизиты (~строка 46) в формате `[Черновик: …]`; `src/partials/footer.html` — три соцссылки (Telegram/VK/YouTube) с `href="#"`. JSON-LD организации (если S09 done) — в `build.js`, `BASE_JSONLD`. Корневые `*.html` — артефакты сборки.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§2 — не выдумывать факты), `src/pages/contacts.html`, `src/partials/footer.html`, `build.js` (BASE_JSONLD, если есть).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
**Блокирован данными владельца.** Запросить: телефон, email, ссылки на реальные соцсети (Telegram/VK/YouTube — какие есть), реквизиты (ИП/самозанятый? ФИО, ИНН, ОГРНИП, юр. адрес). Без данных — не выполнять и не выдумывать; допустимо выполнить частично (только то, что дали) и пометить остальное blocked.

## 4. Затрагиваемые файлы
- `src/pages/contacts.html`
- `src/partials/footer.html`
- `build.js` (дополнить Organization: `telephone`, `email`, `sameAs` — только реальными данными)
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. Контакты: замени заглушки на реальные данные, сделай интерактивными: телефон — `<a href="tel:+7XXXXXXXXXX">+7 (XXX) XXX-XX-XX</a>`, email — `<a href="mailto:...">`, соцсети — прямые ссылки (`https://t.me/...` и т.д.) с `target="_blank" rel="noopener"`. Каналов, которых у владельца нет, — не показывать (убрать колонку/пункт, а не оставлять пустышку).
2. Реквизиты: точный блок по данным владельца (ИП Кнухов И. …, ИНН …, ОГРНИП …). Если владелец не хочет публиковать адрес — согласуй формат (минимум для оферты/политики: наименование + ИНН/ОГРНИП).
3. Футер: соцссылки `href="#"` → реальные URL; иконки несуществующих сетей удалить.
4. JSON-LD (если S09 done): в Organization добавь `telephone`, `email`, `sameAs: [реальные соцссылки]`.
5. `node build.js`; `grep -rn 'Черновик' src/ *.html` — пусто; `grep -n 'href="#"' src/partials/footer.html` — остаются только «Политика»/«Условия», если S19 ещё не сделан (их чинит S19).

## 6. Definition of Done
- [ ] Ни одного «[Черновик:…]» в репозитории.
- [ ] tel:/mailto:/соцссылки кликабельны и верны (сверить посимвольно с данными владельца).
- [ ] В футере нет ссылок-пустышек на соцсети.
- [ ] JSON-LD дополнен только реальными данными.

## 7. Как проверить локально
`node build.js`; `npx serve .` → `/contacts` (клик по tel: на десктопе покажет протокольный диалог — достаточно проверить href в DevTools), футер любой страницы; greps из п. 5.5.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
