# S02 — Кастомная страница 404

## 1. Цель
Сделать фирменную страницу 404 вместо стандартной вercel-овской: удержать заблудившегося посетителя и вернуть его в каталог/на диагностику.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (медицинский консалтинг Ислама Кнухова; аудитория — собственники клиник РФ). Плоский HTML/CSS/JS без фреймворка. Страницы собираются командой `node build.js` из `src/pages/*.html` + `src/partials/{header,footer,scripts-base}.html`; каталог услуг генерируется из массива `SERVICES` в `build.js`. Корневые `*.html` — артефакты сборки, руками не править. Хостинг — Vercel.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md`, `docs/design-guide.md`, `build.js`, `src/pages/cases.html` (образец простой страницы), `styles.css` (классы `.benefits`, `.btn`, `.kicker`, `.lead`).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S01 (чистые URL) — done по STATE.md; ссылки на 404-странице должны быть чистыми.

## 4. Затрагиваемые файлы
- `src/pages/404.html` (создать)
- корневой `404.html` (появится после сборки — Vercel автоматически отдаёт `404.html` для несуществующих путей)

## 5. Точные инструкции
1. Создай `src/pages/404.html` по шаблону страниц проекта (маркеры TITLE/DESCRIPTION/MAIN). Содержание `MAIN` — секция в стиле проекта:
   ```html
   <!--TITLE-->Страница не найдена — Кнухов консалтинг<!--/TITLE-->
   <!--DESCRIPTION-->Такой страницы нет. Перейдите в каталог услуг или пройдите бесплатную экспресс-диагностику клиники.<!--/DESCRIPTION-->
   <!--MAIN-->
   <main id="top">
     <section class="benefits" style="padding-top:64px">
       <div class="container container--wide">
         <div class="benefits_content" style="text-align:center">
           <div>
             <div class="kicker">Ошибка 404</div>
             <h1>Такой страницы нет</h1>
             <p class="lead" style="margin:14px auto 0">Возможно, ссылка устарела. Всё важное — рядом:</p>
           </div>
           <div class="hero-cta" style="justify-content:center">
             <a class="btn btn--dark" href="/uslugi">Каталог услуг</a>
             <a class="btn" href="/diagnostika">Пройти экспресс-диагностику</a>
           </div>
         </div>
       </div>
     </section>
   </main>
   <!--/MAIN-->
   ```
   (h1 здесь корректен; если шаг S05 ещё не сделан — всё равно используй h1.)
2. `node build.js` — файл соберётся автоматически (build.js обрабатывает все `src/pages/*.html`).
3. Если к этому моменту в build.js уже есть генерация sitemap (S07) — исключи `404.html` из sitemap.

## 6. Definition of Done
- [ ] `404.html` в корне собран, использует общие шапку/футер/стили, один h1.
- [ ] Кнопки ведут на `/uslugi` и `/diagnostika`.
- [ ] 404 не попадает в sitemap (если sitemap уже генерируется).

## 7. Как проверить локально
`node build.js`; `npx vercel dev` → открыть несуществующий путь (например `/abc`) → должна показаться фирменная 404. Без vercel dev: открыть `404.html` напрямую через `npx serve .` и проверить вёрстку (десктоп + ≤820px).

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.

## 9. Финализация
Коммит `S02: страница 404`. Обновить `docs/plan/STATE.md` (S02 → done, Журнал). Остановиться.
