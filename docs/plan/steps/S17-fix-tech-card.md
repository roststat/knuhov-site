Проект: knuhov.ru — статический сайт «Кнухов консалтинг». Плоский HTML/CSS/JS, сборка `node build.js` из `src/pages/*.html`. Проблемное место: `src/pages/diagnostika.html`, блок `id="diagnostics"`, третья `.product-card` (~строки 103–115): статическая карточка-ссылка в раздел каталога `/uslugi#tehnicheskie`, внутри пустой `pc-sub` и кнопка «ПЕРЕЙТИ В РАЗДЕЛ». Категории сайта: Диагностика / Сопровождение / Технические работы (массив `CATEGORIES` в `build.js`). Корневые `*.html` — артефакты сборки.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md`, `src/pages/diagnostika.html` (блок diagnostics), `build.js` (CATEGORIES, тексты услуг категории tehnicheskie — для формулировки).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.
