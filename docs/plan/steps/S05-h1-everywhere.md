# S05 — Ровно один h1 на каждой странице

## 1. Цель
Сейчас h1 есть только на `/` и `/diagnostika`; остальные страницы (uslugi, contacts, cases, case-1/2 и все 11 usluga-*) открываются с `<h2>`. Дать каждой странице ровно один h1 — база SEO и доступности.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (медицинский консалтинг Ислама Кнухова). Плоский HTML/CSS/JS. Страницы собираются `node build.js` из `src/pages/*.html`; страницы услуг генерирует функция `servicePageMain(s)` в `build.js` (~строка 176) из массива `SERVICES`. Корневые `*.html` — артефакты сборки, руками не править. Стили заголовков: `styles.css` ~строки 23–26 (h1 clamp 36–60px, h2 clamp 30–42px).
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§6), `docs/design-guide.md`, `build.js`, `src/pages/uslugi.html`, `contacts.html`, `cases.html`, `case-1.html`, `case-2.html`, `styles.css`.
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
Нет. (Если S22 «шаблон услуги v2» уже done — h1 там мог быть решён; сверься со STATE.md и кодом.)

## 4. Затрагиваемые файлы
- `build.js` (`servicePageMain` — `<h2>${s.title}</h2>` → `<h1>`)
- `src/pages/uslugi.html` («Каталог услуг»), `contacts.html` («Как связаться»), `cases.html` («Истории клиник»), `case-1.html`, `case-2.html` («Кейс в подготовке»)
- `styles.css` (при необходимости — модификатор размера)
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. В `build.js` в `servicePageMain` замени `<h2>${s.title}</h2>` на `<h1>${s.title}</h1>`.
2. В каждой из пяти страниц `src/pages/` замени главный (первый, титульный) `<h2>` на `<h1>`, не трогая остальные h2/h3.
3. Визуальный масштаб: страничные заголовки этих внутренних страниц сейчас выглядят как h2 (clamp 30–42px). Чтобы h1 внутренних страниц не стал огромным как на главной, добавь в `styles.css` модификатор и повесь его на заменённые заголовки:
   ```css
   .h1-inner{font-size:clamp(30px,4.2vw,42px);letter-spacing:-1.2px}
   ```
   Использование: `<h1 class="h1-inner">Каталог услуг</h1>` (и в шаблоне услуг).
4. Проверь иерархию после замены: внутри страницы не должно остаться «h3 без h2» из-за переименования. На страницах услуг «Что входит»/«Для кого» — h3; подними их до h2 в `servicePageMain` (и вложенные шаги-строки оставь h3).
5. `node build.js`.

## 6. Definition of Done
- [ ] `for f in *.html; do echo "$f: $(grep -c '<h1' $f)"; done` — ровно 1 на каждой странице (включая 404, если есть).
- [ ] На страницах услуг заголовок — h1, секции — h2, шаги — h3.
- [ ] Визуально размер заголовков внутренних страниц не изменился (или изменился осознанно и аккуратно).

## 7. Как проверить локально
`node build.js`; `npx serve .` → пройти /uslugi, /contacts, /cases, /case-1, любую /usluga-* на десктопе и ≤820px; команда подсчёта h1 из DoD.
