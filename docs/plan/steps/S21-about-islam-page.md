# S21 — Страница «Об Исламе»

## 1. Цель
Бренд эпонимный, продажа идёт через личность, брендовый запрос «Ислам Кнухов» без посадочной. Создать страницу /ob-islame: позиция и манифест (не биография-резюме), фото, связка с диагностикой.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (Ислам Кнухов — стратег по развитию медицинских проектов, основатель; голос «я» — про метод и мышление, «мы» — про исполнение; слово «агентство» запрещено). Плоский HTML/CSS/JS, сборка `node build.js` из `src/pages/*.html`. Готовые примитивы: `.manifesto`/`.manifesto--center` (цитата на чёрном), `.trust` (фото+цифры), `.benefits`, `.audit-steps`, `.btn`. Фото есть: `assets/islam.jpg` (или islam.webp после S15). Позиционирование и тезисы — в `docs/TZ.md` §1, §9а (манифест «почему маркетинг клиник не работает» — текст должен дать владелец).
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§7 — 3-полосная сетка, голос), `docs/design-guide.md`, `docs/TZ.md` (§1), `src/pages/index.html` (блоки «Позиция», «Опыт» — переиспользовать паттерны), `styles.css`.
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S05, S06 — done (правила h1/title). Контент: **запросить у владельца** манифест/тезисы и 3–5 фактов пути (где работал с клиниками, с чего начался подход). До получения — можно собрать страницу из уже подтверждённых материалов сайта (цитата, цифры после S20, метод по слоям) с честными формулировками, без выдуманной биографии.

## 4. Затрагиваемые файлы
- `src/pages/ob-islame.html` (создать)
- `src/partials/header.html`, `src/partials/footer.html` (пункт «Об Исламе» в навигацию — согласуй позицию: между «Кейсы» и «Контакты»)
- `build.js` (JSON-LD: на этой странице Person можно дополнить `mainEntityOfPage`/`url: SITE_URL + '/ob-islame'` — через extraHead-механизм; иначе просто дополни BASE_JSONLD Person полем `url`)
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. TITLE: `Ислам Кнухов — стратег по росту медицинского бизнеса`; DESCRIPTION 140–160 знаков. h1: `Ислам Кнухов` (+ подзаголовок-lead с позицией).
2. Структура MAIN (из готовых примитивов, выравнивание по 3-полосной сетке): хиро-блок с фото (паттерн `.trust-grid`: фото слева 1 полоса, текст позиции 2 полосы) → манифест (`.manifesto--center`, цитата) → «Как я смотрю на клинику» (6 слоёв — переиспользуй формулировки theory-list из `src/pages/diagnostika.html`, вёрстка `.audit-steps` или `.layerstack`) → цифры опыта (блок `.trust-stats`, значения только после S20) → CTA на `/diagnostika`.
3. Голос — «я» (это страница мышления); «мы» — только если абзац про команду.
4. Навигация: добавь «Об Исламе» в `.nav` шапки, мобильное меню (если S16 done) и `.fnav` футера.
5. `node build.js`. Страница попадёт в sitemap (S07) автоматически.

## 6. Definition of Done
- [ ] `/ob-islame` собрана: один h1, уникальные title/description, фото с width/height/alt.
- [ ] Только подтверждённые факты (цитата, слои, цифры после S20, материалы владельца); никакой выдуманной биографии.
- [ ] Пункт меню появился в шапке (десктоп+мобайл) и футере на всех страницах.
- [ ] Секции выровнены по 3-полосной сетке; ч/б; голос «я».

## 7. Как проверить локально
`node build.js`; `npx serve .` → `/ob-islame` на десктопе и 375px; переходы из шапки/футера; проверить sitemap.xml на наличие URL.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
