# S06 — Уникальные title/description + развести дубль кейсов

## 1. Цель
Усилить title/description всех страниц под ключевые запросы («…для клиники», «медицинский консалтинг») и убрать полный дубль `/case-1`–`/case-2` (одинаковые title, description и контент).

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (консалтинг для клиник Ислама Кнухова; SEO-приоритет — Яндекс + Google). Плоский HTML/CSS/JS. title/description каждой страницы живут в маркерах `<!--TITLE-->`/`<!--DESCRIPTION-->` файлов `src/pages/*.html`; у страниц услуг title формируется в `build.js` (~строка 213: `s.title + ' — Кнухов консалтинг'`, description = `s.teaser`). Корневые `*.html` — артефакты сборки.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§6 — формулы title/description), `docs/audit/SEO.md` (§J — карта кластеров), `build.js`, все `src/pages/*.html`.
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S05 — done (h1 расставлены; title согласуются с h1 по смыслу).

## 4. Затрагиваемые файлы
- `src/pages/*.html` (маркеры TITLE/DESCRIPTION)
- `build.js`: в объекты `SERVICES` добавь опциональные поля `seoTitle` и `seoDescription`; при сборке страницы услуги использовать `s.seoTitle || (s.title + ' — Кнухов консалтинг')` и `s.seoDescription || s.teaser`. Заполни поля для всех 11 услуг.
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. Формулы из CONVENTIONS §6: title = `{Суть с ключом} — Кнухов консалтинг`; description 140–160 знаков: ключ + выгода + призыв. Не выдумывать факты (цифры, гарантии).
2. Ориентиры (отредактируй по месту, сохрани голос бренда):
   - `/` — title: `Медицинский консалтинг для клиник — Кнухов консалтинг`; description про «находим, где клиника теряет деньги, и достраиваем систему роста: диагностика, сопровождение, техника».
   - `/uslugi` — `Услуги для роста клиники: диагностика, сопровождение, настройка — Кнухов консалтинг`.
   - `/diagnostika` — `Экспресс-диагностика клиники бесплатно + стратегический аудит — Кнухов консалтинг`.
   - `/contacts` — `Контакты Кнухов консалтинг — заявка на аудит клиники` + description с призывом оставить заявку.
   - `/cases` — `Кейсы: как клиники растят выручку — Кнухов консалтинг`.
   - Услуги (`seoTitle`): `Работа с базой пациентов клиники — Кнухов консалтинг`, `Внедрение и настройка CRM для клиники — …`, `Сквозная аналитика для клиники — …`, `IP-телефония для клиники — …`, `Рассылки и напоминания пациентам — …`, `Автообзвоны пациентов — …`, `Проектный менеджер для клиники на аутсорсе — …`, `Реклама для клиники: специалист на аутсорсе — …`, `HR для клиники: подбор и адаптация персонала — …`, `SMM для клиники — …`, `Тайный покупатель для клиники — …`. `seoDescription` — разверни teaser до 140–160 знаков с выгодой и призывом.
3. Дубль кейсов: в `src/pages/case-1.html` и `case-2.html` сделай title/description и h1 различными (например «Кейс №1 (в подготовке)…» / «Кейс №2 (в подготовке)…», разные description). Плюс, пока кейсы — заглушки, добавь обеим страницам `<meta name="robots" content="noindex,follow">`: расширь `assemble()`/`HEAD()` в build.js опцией `noindex` и передавай её только для case-1/case-2 (механизм: третий маркер `<!--NOINDEX-->` в файле страницы, который build.js превращает в meta). Не забудь снять noindex в шагах S26/S27, когда появятся реальные кейсы.
4. `node build.js`.

## 6. Definition of Done
- [ ] Все title и description уникальны: `grep -h '<title>' *.html | sort | uniq -d` — пусто; то же для description.
- [ ] Каждый description 120–170 знаков, содержит тематический ключ.
- [ ] case-1/case-2 имеют noindex и разные заголовки.
- [ ] У страниц услуг title/description идут из данных `SERVICES`, не из шаблонной склейки.

## 7. Как проверить локально
`node build.js`; команды из DoD; выборочно открыть 3–4 страницы и проверить, что h1 и title не противоречат друг другу.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
