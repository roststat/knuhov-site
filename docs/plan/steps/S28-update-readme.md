# S28 — Актуализировать README.md репозитория

## 1. Цель
README описывает несуществующий сайт (produkty.html, avtozabota.html, инлайн-CSS, logo.png) и собьёт с толку любую будущую сессию. Переписать под фактическую архитектуру.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг». Фактическая архитектура: `node build.js` собирает `src/pages/*.html` + `src/partials/{header,footer,scripts-base}.html` в корневые `.html`; каталог услуг и страницы `/usluga-*` генерируются из массива `SERVICES` в `build.js`; общий `styles.css`; деплой Vercel (`vercel --prod --yes`), домен через VPS-прокси. Планирование работ — `docs/plan/` (README/CONVENTIONS/STATE/steps), аудит — `docs/audit/`.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md`, текущий `README.md` (что устарело), `build.js`, `ls` корня и `src/`.
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.
