# S12 — Комплект ч/б-изображений для карточек услуг (11 шт)

## 1. Цель
Подготовить единый по стилю комплект изображений для визуалов карточек каталога (сейчас `card-visual` — пустой прямоугольник с точкой). Только ассеты и согласование стиля; подключение в разметку — шаг S13.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (консалтинг для клиник). Бренд строго ч/б (#000/#fff, Space Grotesk, фирменный нижний-правый угол). Карточки каталога рендерит `cardHTML(s)` в `build.js`; в каждой есть `<div class="card-visual">` с `aspect-ratio:4/3` (`styles.css`). Услуги (11 со slug): rabota-s-bazoy, proektnyy-menedzher, reklamshchik, hr, smm, taynyy-pokupatel, crm, skvoznaya-analitika, telefoniya, rassylki, avtoobzvony — список и суть смотри в массиве `SERVICES` в `build.js`.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§4 — стиль/форматы/именование), `docs/design-guide.md`, `build.js` (SERVICES), `styles.css` (`.card-visual`).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
Формально нет, но **нужно одобрение владельца по стилю** (см. п. 5.1). Без ответа владельца шаг можно довести до варианта Б (графика) — он не требует внешних фото.

## 4. Затрагиваемые файлы
- `assets/usluga-<slug>.webp` × 11 (создать)
- при варианте с фото — временная папка исходников в scratch (в репо не коммитить)

## 5. Точные инструкции
1. Предложи владельцу 2 варианта единого стиля (покажи по 1 образцу каждого):
   - **А. Фото-дуотон:** стоковые/собственные фото медицинской тематики (ресепшн, врач, телефония, дашборд), приведённые к ч/б: `magick in.jpg -colorspace Gray -level 5%,95% -resize 800x600^ -gravity center -extent 800x600 -quality 80 out.webp`. Источник фото согласовать (лицензия!) — без одобрения не брать.
   - **Б. Строгая линейная графика (рекомендуемый дефолт, если владелец молчит):** одноцветные чёрные пиктограммы-композиции на белом в стилистике мозаики бренда (толстая линия 2px, скруглённый нижний-правый угол у рамочных элементов) — рисуются как SVG прямо в этой сессии, без внешних зависимостей, далее рендер в webp 800×600 или подключение как SVG (тогда именование `assets/usluga-<slug>.svg`, отметить в STATE для S13).
2. Сделай по одному изображению на каждую из 11 услуг; метафора — из teaser услуги (база → сегменты/карточки пациентов, телефония → трубка/волны, CRM → воронка, аналитика → связанные графики, тайный покупатель → маска/чек-лист и т.п.).
3. Требования (CONVENTIONS §4): пропорция 4/3 (800×600), строго ч/б, вес ≤ 60 КБ каждый, именование `assets/usluga-<slug>.webp|svg`.
4. Никаких цветных вставок; никаких стоковых фото без подтверждённой лицензии.

## 6. Definition of Done
- [ ] 11 файлов в `assets/` с правильными именами, 4/3, ч/б, ≤ 60 КБ.
- [ ] Стиль един для всех 11 (один визуальный язык).
- [ ] Вариант стиля зафиксирован в STATE.md-заметке (А или Б; если А — источник и лицензия).

## 7. Как проверить локально
Открыть все 11 файлов (Quick Look/браузер) рядом — единый стиль; `du -h assets/usluga-*` — веса в норме. Сайт на этом шаге не меняется, сборка не нужна.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.

## 9. Финализация
Коммит `S12: изображения карточек услуг (11 шт)`. Обновить `docs/plan/STATE.md` (S12 → done + заметка о выбранном стиле/формате, Журнал). Остановиться.
