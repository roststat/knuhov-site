# S22 — Расширенный шаблон страницы услуги (процесс, FAQ, смежные услуги)

## 1. Цель
Страницы услуг сейчас тонкие (~120 слов: pain + 5 пунктов + forWhom) и тупиковые. Расширить ШАБЛОН в build.js новыми опциональными секциями: «Как проходит работа», FAQ, «Смежные услуги» + CTA на диагностику. Тексты наполняются следующими шагами (S23/S24) — здесь только механика и вёрстка.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг». Плоский HTML/CSS/JS. Страницы `/usluga-*` генерирует функция `servicePageMain(s)` в `build.js` из объектов массива `SERVICES` (поля: cat, slug, title, teaser, pain, includes[], forWhom; после S06 ещё seoTitle/seoDescription; после S13 — image). Правило проекта: контент услуг живёт в данных, разметка — в шаблоне. Вёрстка из готовых примитивов: `.audit-steps`/`.step-row` (нумерованные шаги), `.benefits_content`, `.product-card`, `.btn`. Корневые `*.html` — артефакты сборки.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§2, §7), `docs/design-guide.md` (§12), `build.js` целиком, `styles.css`, собранный `usluga-crm.html` (текущий вид).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S05 — done (h1 в шаблоне). Совместим с S11 (крошки) и S13 (image) — если они done, не сломай их вставки.

## 4. Затрагиваемые файлы
- `build.js` (`servicePageMain`, опционально `serviceJSONLD` — FAQ-разметка)
- `styles.css` (компонент FAQ, если понадобится)
- корневые `usluga-*.html` — пересборка

## 5. Точные инструкции
1. Новые опциональные поля объекта услуги (документируй комментарием над `SERVICES`):
   - `process: [{t:'Заголовок шага', d:'описание'}, …]` — 3–4 шага «как проходит работа»;
   - `faq: [{q:'Вопрос', a:'Ответ'}, …]` — 3–5 пар;
   - `seoText: 'абзац(ы) через \n\n'` — расширенный текст под заголовком «Подробнее» (опционально).
2. В `servicePageMain(s)` после блока «Для кого» добавь условные секции (рендерятся только при наличии поля):
   - **Процесс:** заголовок h2 «Как проходит работа», вёрстка `.audit-steps` (переиспользуй `stepRows`-подход, но с t/d).
   - **seoText:** h2 «Подробнее об услуге», абзацы `<p class="lead" style="max-width:680px">` (или новый скромный класс).
   - **FAQ:** h2 «Вопросы и ответы», семантика `<details class="faq-item"><summary>q</summary><p>a</p></details>`. Стили: рамка 1px, `--radius-sm`, паддинг 16–18px, маркер-плюс через `summary::after` (ч/б).
   - **Смежные услуги (всегда):** h2 «Смежные услуги», 3 карточки `.product-card` других услуг той же категории (если в категории меньше 3 других — добери из соседней). Генерация из `SERVICES` — переиспользуй `cardHTML(s)`.
   - **CTA (всегда, в конце):** существующий блок «Оставить заявку» дополни второй кнопкой `<a class="btn" href="/diagnostika">Пройти экспресс-диагностику</a>`.
3. FAQ-разметка schema.org: если S10 done — в `serviceJSONLD(s)` при наличии `s.faq` добавь в @graph сущность `{'@type':'FAQPage','mainEntity':[{'@type':'Question',name:q,acceptedAnswer:{'@type':'Answer',text:a}}…]}`.
4. Для проверки механики заполни `process` и `faq` ОДНОЙ услуге (`crm`) черновыми, но правдоподобными текстами на основе её includes (пометь в STATE, что тексты crm — черновик до S24).
5. `node build.js`.

## 6. Definition of Done
- [ ] Услуга без новых полей рендерится как раньше (регресс-проверка: `usluga-hr.html`).
- [ ] `usluga-crm.html` показывает процесс, FAQ (details раскрываются), смежные услуги, двойной CTA.
- [ ] Смежные карточки не включают текущую услугу и ведут на существующие страницы.
- [ ] Иерархия h1→h2→h3 не нарушена; всё ч/б, выравнивание по 3-полосной сетке.

## 7. Как проверить локально
`node build.js`; `npx serve .` → `/usluga-crm` (полный шаблон) и `/usluga-hr` (старый вид) на десктопе и 375px; клавиатура: Tab до summary, Enter раскрывает.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
