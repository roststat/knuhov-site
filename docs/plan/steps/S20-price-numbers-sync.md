# S20 — Сверка цены аудита и цифр опыта с владельцем

## 1. Цель
Устранить расхождение фактов: ТЗ (`docs/TZ.md`) называет цену стратегического аудита 50 000 ₽, сайт показывает «От 25 000 ₽»; цифры блока «Опыт» (100 аудитов / 20 клиник / 5 в портфеле) нигде не подтверждены (README называет цифры сайта плейсхолдерами). Получить решение владельца и привести сайт к одной правде.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг». Плоский HTML/CSS/JS, сборка `node build.js`. Места фактов: цена аудита — `build.js` (`SERVICES`, услуга «Стратегический аудит», поле `price`) и `src/pages/diagnostika.html` (карточка `#audit-card`, `.pc-price`); цифры опыта — `src/pages/index.html` и `src/pages/diagnostika.html`, блоки `.trust-stats` (`data-count="100|20|5"`). Корневые `*.html` — артефакты сборки.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§2 — не выдумывать факты), `docs/TZ.md` (строка про 50 000 ₽), `build.js` (SERVICES), `src/pages/index.html`, `src/pages/diagnostika.html`.
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
**Блокирован ответами владельца.** Вопросы задать дословно: 1) Какая актуальная цена стратегического аудита и в каком формате её показывать («От 25 000 ₽» / «50 000 ₽» / «по запросу»)? 2) Верны ли цифры: 100 проведённых аудитов, 20 клиник внедрили систему, 5 в портфеле сейчас? Если нет — какие верны? 3) Действует ли условие «стоимость аудита засчитывается в первый чек» (упомянуто в ТЗ) — показывать ли его на сайте?

## 4. Затрагиваемые файлы
- `build.js` (price аудита)
- `src/pages/diagnostika.html` (цена в карточке аудита; при необходимости — строка про зачёт в первый чек)
- `src/pages/index.html`, `src/pages/diagnostika.html` (data-count и подписи в .trust-stats)
- `docs/TZ.md` НЕ править (исторический документ)
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. Получи ответы владельца (см. п. 3). Ничего не менять до ответов.
2. Цена: обнови оба места одинаково (данные `SERVICES` + карточка на diagnostika). Формат «От N ₽» или точная сумма — как решил владелец.
3. Цифры опыта: обнови `data-count` и подписи в обоих файлах синхронно. Если владелец просит убрать какую-то метрику — убери весь `.trust-stat`, сетка `repeat(3,1fr)` перейдёт на фактическое число колонок только после правки CSS — при удалении элемента поправь `grid-template-columns` соответствующего блока или оставь 3 метрики, заменив спорную на подтверждённую.
4. Зачёт в первый чек (если владелец подтвердил): добавь строку в карточку аудита (`pc-sub` дополнить) или в блок «Что происходит на стратегическом аудите» — одним предложением, без новых секций.
5. `node build.js`; `grep -rn '25 000\|50 000' src/ build.js` — только согласованное значение; `grep -rn 'data-count' src/` — согласованные цифры.

## 6. Definition of Done
- [ ] Цена аудита одинакова во всех местах сайта и подтверждена владельцем.
- [ ] Цифры опыта подтверждены (или заменены/убраны) — на обеих страницах синхронно.
- [ ] Ответы владельца зафиксированы в заметке STATE.md (чтобы будущие сессии не переспрашивали).

## 7. Как проверить локально
`node build.js`; `npx serve .` → `/diagnostika` (карточка аудита), `/uslugi` (карточка «Стратегический аудит»), блоки «Опыт» на `/` и `/diagnostika`; greps из п. 5.5.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
