# S26 — Кейс №1 с цифрами

## 1. Цель
Заменить заглушку «Кейс в подготовке» на первый реальный обезличенный кейс по формуле «что нашли → что достроили → что изменилось» — главный недостающий блок доверия сайта.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (консалтинг для клиник; кейсы — обезличенные разборы, «мышление Ислама в действии», не логотипы и звёзды — см. docs/TZ.md §4 блок 7, §9а). Плоский HTML/CSS/JS, сборка `node build.js`. Текущие заглушки: `src/pages/case-1.html` (структура 01-Задача/02-Что сделали/03-Результат на `.audit-steps`), карточки-тизеры на `src/pages/index.html` (блок `#cases-teaser`) и `src/pages/cases.html`. После S06 у case-страниц стоит `noindex` (маркер в файле страницы) — реальному кейсу его надо снять.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§2 — только факты владельца), `docs/TZ.md` (§4 блок 7), `src/pages/case-1.html`, `src/pages/cases.html`, `src/pages/index.html` (блок cases-teaser), `styles.css` (`.audit-steps`, `.trust-stats`, `.stat`).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
**Блокирован материалами владельца.** Запросить: тип клиники (стоматология/многопрофильная…, город — можно без названия), исходная ситуация и что было не так; что нашли на диагностике/аудите (по слоям); что внедрили; результат в цифрах (выручка/записи/возвраты, «до → после», период); подтверждение, что клиника согласовала публикацию в обезличенном виде.

## 4. Затрагиваемые файлы
- `src/pages/case-1.html` (полная замена контента; переименовывать файл в смысловой слаг, напр. `case-stomatologiya.html`, МОЖНО по желанию владельца — тогда обнови все ссылки: index, cases, sitemap соберётся сам)
- `src/pages/cases.html`, `src/pages/index.html` (карточка-тизер: реальный заголовок, суть, ключевая цифра)
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. Структура страницы кейса (из готовых примитивов): h1 — суть кейса с результатом (без названия клиники); лид — тип клиники и исходная ситуация; блок «Что нашли» (слои, где текло — `.audit-steps` или `.qlist`); «Что достроили» (шаги внедрения); «Что изменилось» — цифры «до → после» (используй `.trust-stats`/`.stat` для крупных цифр); честная оговорка о периоде и контексте; CTA на `/diagnostika`.
2. Все цифры и факты — дословно из материалов владельца, обезличивание — проверить (никаких названий, имён врачей, узнаваемых деталей).
3. Снять noindex с этой страницы (убрать маркер из файла); title/description уникальные, с результатом в title.
4. Тизеры на `/` и `/cases`: замени первую карточку «Готовим кейс» на реальную (тег категории/типа клиники, заголовок с цифрой, ссылка на страницу кейса).
5. `node build.js`; проверь, что страница попала в sitemap (noindex снят).

## 6. Definition of Done
- [ ] Страница кейса: один h1, формула «нашли → достроили → изменилось», цифры «до → после», без идентифицирующих деталей.
- [ ] Тизеры на главной и /cases ведут на кейс и не дублируют друг друга дословно.
- [ ] noindex снят, страница в sitemap; факты подтверждены владельцем (заметка в STATE).

## 7. Как проверить локально
`node build.js`; `npx serve .` → страница кейса (десктоп/375px), переходы с `/` и `/cases`; `grep noindex <файл>` — пусто; sitemap.xml содержит URL.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.

## 9. Финализация
Коммит `S26: кейс №1`. Обновить `docs/plan/STATE.md` (S26 → done + заметка «факты согласованы владельцем [дата]», Журнал). Остановиться.
