# S27 — Кейс №2 с цифрами

## 1. Цель
Второй реальный обезличенный кейс — по образцу и структуре кейса №1 (S26). Два кейса закрывают блок «С кем уже работали» на главной полностью реальным контентом.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг». Плоский HTML/CSS/JS, сборка `node build.js`. Кейс №1 уже опубликован (S26) — его страница задаёт структуру и стиль («что нашли → что достроили → что изменилось», цифры «до → после», обезличенность). Текущая заглушка: `src/pages/case-2.html`; тизеры — вторые карточки в `src/pages/index.html` (#cases-teaser) и `src/pages/cases.html`.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§2), страницу кейса №1 в `src/pages/` (эталон), `src/pages/case-2.html`, `src/pages/cases.html`, `src/pages/index.html`.
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S26 — done (эталон структуры). **Блокирован материалами владельца** (тот же список вопросов, что в S26 п. 3). Желательно кейс из другой категории работ (например, если №1 про сопровождение — №2 про технические работы), чтобы покрыть разные услуги.

## 4. Затрагиваемые файлы
- `src/pages/case-2.html` (полная замена; опционально переименование в смысловой слаг с обновлением ссылок)
- `src/pages/cases.html`, `src/pages/index.html` (вторая карточка-тизер)
- корневые `*.html` — пересборка

## 5. Точные инструкции
1. Скопируй структуру страницы кейса №1, наполни материалами владельца (правила обезличивания и честности — как в S26 п. 5.1–5.2).
2. Свяжи кейс с релевантными услугами: в тексте «что достроили» дай ссылки на 1–2 страницы `/usluga-*`, которые соответствуют внедрённому.
3. Сними noindex, уникальные title/description с результатом.
4. Обнови вторые тизер-карточки на `/` и `/cases`.
5. Если на `/cases` остались только 2 кейса — проверь, что лид секции больше не говорит «раздел в наполнении» (поправь формулировку на нейтральную).
6. `node build.js`.

## 6. Definition of Done
- [ ] Страница кейса №2 в структуре эталона, с цифрами и перелинковкой на услуги.
- [ ] Заглушек «Готовим кейс» на сайте не осталось; лид /cases актуален.
- [ ] noindex снят, страница в sitemap; факты подтверждены владельцем.

## 7. Как проверить локально
`node build.js`; `npx serve .` → `/cases` и оба кейса (десктоп/375px); ссылки на услуги из текста кейса работают; `grep -rn 'Готовим кейс' src/` — пусто.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
