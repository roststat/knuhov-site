# S25 — FAQ-блок на /diagnostika + разметка FAQPage

## 1. Цель
Снять типовые возражения перед лид-магнитом (бесплатный тест) и платным аудитом: видимый FAQ-блок на /diagnostika + разметка FAQPage для сниппетов в выдаче.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (лид-магнит — бесплатный тест из 6 вопросов в попапе на /diagnostika, платный продукт — стратегический аудит 2–3 часа лично с Исламом). Плоский HTML/CSS/JS, сборка `node build.js` из `src/pages/*.html`. Компонент FAQ (`<details class="faq-item">` + стили) появился в S22 — переиспользовать. JSON-LD вставляется механизмами из S09/S10.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§6, §7), `src/pages/diagnostika.html` целиком (факты для ответов: 6 вопросов, 2 минуты, без регистрации, слои, формат аудита), `build.js`, `styles.css` (`.faq-item` из S22).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S09 — done (механика JSON-LD). S22 — желательно done (готовый компонент FAQ; иначе создать его здесь по описанию из S22 п. 5.2).

## 4. Затрагиваемые файлы
- `src/pages/diagnostika.html` (секция FAQ перед финальным блоком; JSON-LD FAQPage — в `<!--SCRIPTS-->` страницы отдельным `<script type="application/ld+json">`)
- `styles.css` (только если компонент FAQ ещё не создан)
- корневой `diagnostika.html` — пересборка

## 5. Точные инструкции
1. Составь 5–6 пар вопрос-ответ ТОЛЬКО из фактов, которые уже есть на странице/сайте: «Сколько занимает тест?» (2 минуты, 6 вопросов, без регистрации и контактов), «Что я получу по итогам теста?» (тип системы, слабые слои, гипотезы; можно скачать PDF), «Чем тест отличается от аудита?» (тест — симптомы, аудит — 2–3 часа лично с Исламом и дорожная карта), «Сколько стоит аудит?» (значение после S20 — сверься со STATE), «Нужно ли готовить данные клиники?» (для теста — нет; для аудита — цифры обсуждаются при бронировании; если факта нет — вопрос не включать), «Онлайн или очно?» (из блока «Бронируем время»: онлайн или очно).
2. Вставь секцию после блока `#process` («Что происходит на стратегическом аудите»), в тех же контейнерах: `section.benefits` → `.container container--wide` → h2 «Вопросы и ответы» + `<details class="faq-item">…`. Выравнивание по 3-полосной сетке (ограничь ширину списка, как у `.audit-steps`).
3. JSON-LD: в `<!--SCRIPTS-->` страницы добавь `<script type="application/ld+json">` с `FAQPage` → `mainEntity` из тех же пар (текст в разметке = видимый текст на странице, требование Google/Яндекса).
4. `node build.js`.

## 6. Definition of Done
- [ ] FAQ-блок на /diagnostika: 5–6 details, раскрываются мышью и клавиатурой, ч/б стиль.
- [ ] FAQPage-разметка валидна и дословно совпадает с видимым текстом.
- [ ] Все ответы — подтверждённые факты сайта, без выдумок.
