# S24 — SEO-тексты услуг категории «Технические работы» (5 услуг)

## 1. Цель
Аналогично S23, но для категории «Технические работы»: наполнить 5 страниц (crm, skvoznaya-analitika, telefoniya, rassylki, avtoobzvony) до 300–500 слов через поля process/faq/seoText в `SERVICES`.

## 2. Контекст-бутстрап
Проект: knuhov.ru — статический сайт «Кнухов консалтинг» (консалтинг для клиник; голос «мы» для исполнения; «агентство» запрещено; без выдуманных фактов). Плоский HTML/CSS/JS. Контент услуг — массив `SERVICES` в `build.js`; шаблон страниц услуг поддерживает опциональные поля `process/faq/seoText` (S22); `node build.js` пересобирает. Категория `tehnicheskie`: crm (Настройка CRM), skvoznaya-analitika, telefoniya, rassylki, avtoobzvony. В `docs/TZ.md` §9 есть черновой список конкретных систем (МИС/CRM: YCLIENTS, amoCRM, Битрикс24, IDENT, Инфоклиника…; телефония: Телфин; аналитика: Roistat, Calltouch, Я.Метрика) — использовать как «работаем с любыми системами, в том числе…», список финализирует владелец.
Прочитай первыми: `docs/plan/README.md`, `docs/plan/CONVENTIONS.md` (§2, §6), `docs/audit/SEO.md` (§J), `docs/TZ.md` (§9), `build.js`, собранный `usluga-rabota-s-bazoy.html` (образец после S23).
Глоссарий: «пуш» (команда владельца) = git add+commit → `git push origin main` → `vercel --prod --yes`. Без команды «пуш» не деплоить.

## 3. Предусловия/зависимости
S22 — done. S23 — желательно done (единый тон между категориями; если S23 ещё не сделан — этот шаг можно выполнять первым, тогда он задаёт тон).

## 4. Затрагиваемые файлы
- `build.js` (только данные `SERVICES` — 5 объектов категории tehnicheskie; у `crm` заменить черновые process/faq из S22 на чистовые)
- корневые `usluga-crm.html`, `usluga-skvoznaya-analitika.html`, `usluga-telefoniya.html`, `usluga-rassylki.html`, `usluga-avtoobzvony.html` — пересборка

## 5. Точные инструкции
1. Заполни process/faq/seoText для 5 услуг (правила и структура — как в S23 п. 5.1–5.2).
2. Специфика категории: в seoText уместно упомянуть классы систем и примеры из TZ §9 в формате «работаем с любыми: в том числе X, Y, Z» — это охват запросов «внедрение amoCRM в клинике», «IDENT настройка» без привязки к вендору. Не обещать партнёрств/сертификаций, которых нет.
3. Типовые FAQ категории: «Какая CRM/МИС подойдёт нашей клинике?», «Сколько длится внедрение?», «Что нужно от нас?», «Не сломается ли текущий процесс?» (ответ в духе «ничего не ломаем, достраиваем» — формула бренда из TZ §4/блок 6), «Как это связано с выручкой?».
4. `node build.js`; вычитать 2 страницы глазами.

## 6. Definition of Done
- [ ] Все 5 услуг категории имеют process + faq + seoText; страницы 300–500 слов.
- [ ] Названия систем — только как примеры охвата, без ложных обещаний.
- [ ] Черновой FAQ crm из S22 заменён чистовым.
- [ ] Правки только в данных SERVICES.

## 7. Как проверить локально
`node build.js`; `npx serve .` → все 5 страниц (десктоп + 375px); details-FAQ раскрываются; объём ≥ 300 слов.

## 8. Деплой
По умолчанию НЕ деплоить: изменения уходят пачкой по команде владельца «пуш». Если скомандовал: `git push origin main && vercel --prod --yes`.
