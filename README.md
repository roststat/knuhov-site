# Кнухов консалтинг — сайт

Статический многостраничный сайт knuhov.ru: медицинский консалтинг Ислама Кнухова. Без фреймворка — плоский HTML/CSS/JS, собирается Node-скриптом. Дизайн-система: строгий ч/б-каркас (кнопки, рамки, типографика) + цветные иллюстрации (в процессе пересмотра стиля), фирменный приём «один скруглённый угол» (нижний-правый), один гротеск.

## Как собрать

```
node build.js
```

Скрипт склеивает `src/pages/*.html` + `src/partials/{header,footer,scripts-base}.html` в обычные `.html` в корне репозитория и генерирует:
- `robots.txt` и `sitemap.xml` — из фактически собранных страниц;
- страницы услуг `usluga-<slug>.html` — из массива `SERVICES` в `build.js`.

**Корневые `*.html`, `robots.txt`, `sitemap.xml` — артефакты сборки.** Их не редактируют руками: правки идут в `src/pages/`, `src/partials/` или `build.js`, затем заново `node build.js`. Собранные файлы коммитятся вместе с исходниками — Vercel раздаёт корень репозитория как есть.

## Структура

| Путь | Назначение |
|---|---|
| `build.js` | Сборщик: HEAD-шаблон (canonical, OG, JSON-LD, favicon), каталог услуг (`SERVICES`, `CATEGORIES`), генерация страниц услуг, sitemap |
| `src/pages/*.html` | Исходники страниц (маркеры `<!--TITLE-->`, `<!--DESCRIPTION-->`, `<!--MAIN-->`, опц. `<!--SCRIPTS-->`, `<!--NOINDEX-->`) |
| `src/partials/header.html` | Шапка: лого, десктоп-навигация, мобильный бургер + панель |
| `src/partials/footer.html` | Подвал |
| `src/partials/scripts-base.html` | Сквозной JS: `window.track()`, бургер-меню, счётчик цифр, попапы |
| `styles.css` | Единый файл стилей, вся дизайн-система и примитивы (см. `docs/design-guide.md`) |
| `assets/` | Изображения и иллюстрации |
| `docs/design-guide.md` | Дизайн-токены и переиспользуемые классы |
| `docs/TZ.md` | Исходное ТЗ (историческое, не редактируется) |
| `docs/audit/` | SEO/UX-аудит сайта |
| `docs/plan/` | План работ: `README.md` (оглавление), `CONVENTIONS.md` (единые решения), `STATE.md` (прогресс — источник правды между сессиями), `steps/*.md` (атомарные шаги) |

## Как добавить услугу

Добавить объект в массив `SERVICES` в `build.js` — карточка в каталоге `/uslugi` и отдельная страница `/usluga-<slug>` сгенерируются автоматически. Поля объекта:

- `cat` — id категории (`diagnostika` / `soprovozhdenie` / `tehnicheskie`);
- `slug` — латиница-kebab-case, определяет URL страницы; услуги без `slug` (используют `href`) своей страницы не получают;
- `title`, `teaser`, `price` (опц.), `pain`, `includes[]`, `forWhom` — контент карточки/страницы;
- `seoTitle`, `seoDescription` (опц.) — переопределяют дефолтные title/description;
- `image` (опц.) — путь к иллюстрации; без поля используется конвенция `/assets/usluga-<slug>.svg`;
- `process[{t,d}]`, `faq[{q,a}]`, `seoText` (опц.) — расширенные секции страницы услуги («Как проходит работа», FAQ + разметка FAQPage, «Подробнее об услуге»).

## Как добавить страницу

Создать файл в `src/pages/` с маркерами `<!--TITLE-->`/`<!--DESCRIPTION-->`/`<!--MAIN-->` (опционально `<!--SCRIPTS-->` для инлайн-скриптов страницы, `<!--NOINDEX-->` — чтобы исключить из индексации и sitemap). `node build.js` подхватит файл автоматически.

## Дизайн-система (кратко)

- Фирменный угол: только нижний-правый, 24/12/8px (`--radius`/`--radius-sm`/`--radius-xs`), везде остальные углы — 0.
- Шрифт: Space Grotesk (Google Fonts CDN). **Известное ограничение:** у Space Grotesk нет кириллицы — весь русский текст рендерится системным шрифтом-фолбэком; решение о переходе на другой гротеск ждёт владельца.
- Меню: `position:fixed`, высота 70px (`--header-h`), на ≤820px — бургер-меню (`src/partials/header.html` + стили в `styles.css`).
- Цвет: строгий ч/б-каркас (кнопки, рамки, типографика) остаётся неизменным; иллюстрации/графика — цветные (решение владельца от 2026-07-05), но стиль текущего комплекта не утверждён и будет пересматриваться — см. `docs/plan/STATE.md`.
- Полная библиотека примитивов и токенов — `docs/design-guide.md`.

## Аналитика

Слой событий уже готов: `window.track(name, payload)` в `src/partials/scripts-base.html` пушит в `dataLayer` и логирует в консоль. Реальный счётчик (Яндекс.Метрика) пока не подключён — см. шаг `docs/plan/steps/S32-yandex-metrika.md`.
